What happens when you initialize a repository? Why do you need to do it?

	It creates the git metadata to make it a git repository. It's necessary in order for git to be able to track the changes.

How is the staging area different from the working directory and the repository?
What value do you think it offers?

	The staging area allows you to setup a potential commit. It gives the flexibility of choosing which files will be committed.

How can you use the staging area to make sure you have one commit per logical
change?

	You can add just the files you want committed and leave the others out.

What are some situations when branches would be helpful in keeping your history
organized? How would branches help?

	Branches can help you try different methods or features without harming existing work.

How do the diagrams help you visualize the branch structure?

	It illustrates the connections between branches and the commits that make them up.

What is the result of merging two branches together? Why do we represent it in
the diagram the way we do?

	It combines the lines from both branches. Because it shows the two different paths and how they come together.

What are the pros and cons of Git's automatic merging vs. always doing merges
manually?

	With simple changes automatic merge makes it easy. The downside is that you often have to intervene because it's low intelligence.
